## 1. 目标

### 完成定义

`/refactor skill {path}` 可对任意 SKILL.md 执行完整重构——术语统一、冗余消除、结构规范化——产出直接可用，无需人工二次修改。

### 价值目标

用户可以用一条命令将粗糙的 skill 文档重构为符合结构规范的成熟版本。

### 关键假设

- **假设**: AI 按照 refactor skill 的规则集执行重构，产出质量稳定高于人工手工重构
- **类型**: 技术可行性
- **验证方式**: 对 ≥2 个真实 skill 执行 `/refactor skill`，对比重构前后的结构完整度和术语一致性，产出可直接使用无需大幅修改

## 2. 计划

### 预期交付物

- 文档: 打磨后的 refactor skill SKILL.md（规则完善、边界明确）
- 数据: ≥2 个真实 skill 的重构案例记录（before/after 对比）

### 达标标准

| 维度 | 标准 |
|------|------|
| 功能完整性 | 对 ≥2 个真实 skill 完成重构，产出符合目标结构（经验值: 所有必需 section 存在） |
| 质量 | 重构产出可直接使用，用户无需大幅修改（经验值: 修改 <10 行即可投入使用） |
| 性能与稳定性 | 本里程碑不涉及 |
| 安全 | 本里程碑不涉及 |
| 可维护性 | refactor skill 规则集无矛盾、无歧义，新贡献者读完可独立执行重构 |

### 预估投入

2 天全职开发（1 天打磨规则 + 1 天验证案例）

### 决策路径

- **通过**: 进入 M2 脚手架创建能力开发，refactor skill 的规则集可作为 create 子命令的结构模板基础
- **未通过**: 分析重构失败案例，补充缺失规则或修正冲突规则，暂不启动 M2

### 风险

风险: skill 结构规范尚未完全标准化，重构规则可能在案例验证中暴露不足需要迭代

## 1. 目标

### 完成定义

`/refactor create {name}` 可生成符合结构规范的 skill 骨架——包含 SKILL.md、plugin.json、目录结构——用户在骨架基础上填写内容即可使用。

### 价值目标

用户可以用一条命令创建结构完整的新 skill，不再从零开始或从其他 skill 复制修改。

### 关键假设

- **假设**: 提供结构化脚手架比从已有 skill 复制修改能减少遗漏和残留内容
- **类型**: 用户接受度
- **验证方式**: 用 `/refactor create` 创建 1 个新 skill，对比脚手架产出与手工复制修改的结构完整度和残留内容数

## 2. 计划

### 预期交付物

- 代码: `refactor create` 子命令实现
- 文档: SKILL.md 中 create 子命令的路由和工作流定义

### 达标标准

| 维度 | 标准 |
|------|------|
| 功能完整性 | `/refactor create {name}` 生成可用骨架，包含 SKILL.md（含必需 section）+ plugin.json + 目录结构 |
| 质量 | 骨架产出无残留占位符（{xxx} 均有注释提示），用户只需填写内容 |
| 性能与稳定性 | 本里程碑不涉及 |
| 安全 | 本里程碑不涉及 |
| 可维护性 | 骨架模板与 M1 的 refactor 目标结构保持一致，修改一处规范两处同步 |

# M2 — 建立 simulate 行为链验证能力

## 1. 目标

### 完成定义

`/skill-wrapper simulate {skill_path} {input}` 可对任意 skill + 模拟输入进行 dry-run，输出 stage+step+gate 级别的行为链，规则冲突标记 [AMBIGUOUS]。

### 价值目标

用户可以在发布前预览 AI 拿到 skill 后会怎么执行，不再靠经验猜测。

### 关键假设

- **假设**: AI 可以基于 skill 规则准确推理出执行行为链，且推理结果与实际执行一致
- **类型**: 技术可行性
- **验证方式**: 对 ≥1 个真实 skill 执行 simulate，然后实际执行同一输入，对比行为链一致性

## 2. 计划

### 预期交付物

- 文档: SKILL.md 中 simulate 子命令定义（workflow + 行为链模板）
- 数据: ≥1 个真实 skill 的 simulate vs 实际执行对比

### 达标标准

| 维度 | 标准 |
|------|------|
| 功能完整性 | 行为链覆盖 Input Normalization → Evaluate → 每个 stage 的 step+gate 判定 |
| 质量 | 行为链与实际执行一致率 ≥80%（gate 判定和 stage 选择与实际一致） |
| 性能与稳定性 | 本里程碑不涉及 |
| 安全 | simulate 不执行任何实际操作（不创建文件、不调用脚本） |
| 可维护性 | 行为链模板可适配不同 skill 结构 |

### 预估投入

2 天全职开发

### 决策路径

- **通过**: v1 闭环——lint + simulate 质量保障能力可用，开始在实际 skill 开发中持续使用
- **未通过**: 分析推理偏差原因——skill 规则歧义导致还是 AI 推理能力限制

### 风险

依赖 M1 lint 规则集（[AMBIGUOUS] 检测复用 lint 的规则冲突分析）；风险：复杂 skill 行为链过长导致推理准确度下降

# skill-wrapper 产品路线图

## 1. 产品愿景

### 产品核心

- **定位**: 面向 Claude Code skill 作者的开发工具集
- **动机**: skill 从编写到成熟需要反复重构、测试、验证，但这些操作没有标准化流程，全靠人工经验和反复试错
- **长远愿景**: skill 作者像软件工程师一样工作——有脚手架创建、有测试验证、有重构工具，skill 的质量从依赖个人经验变为依赖工具链保障

### 价值体系

| 层级 | 价值 | 衡量指标 |
|------|------|---------|
| **即时价值** | skill 的创建、测试、重构每个环节都有对应工具，不再纯手工操作 | 子命令覆盖率（目标值，待验证: 覆盖创建/测试/重构 3 个环节） |
| **累积价值** | 工具链沉淀的最佳实践让新 skill 的起点越来越高 | 待验证（预期: 新 skill 从创建到可用的时间缩短） |
| **战略价值** | skill 开发从手工作坊升级为工程化流程，质量可预期、可复现 | 无数据（尚无量化手段） |

### 核心问题

| 问题 | 发生频率 | 单次成本 | 影响面 | 现有应对 |
|------|---------|---------|--------|---------|
| skill 重构缺乏标准化工具，质量依赖人工判断 | 每次 skill 迭代（估算: 每个 skill 生命周期 3-5 次，基于作者经验） | 估算 30-60 分钟手工审查+修改（基于作者体感） | 所有 Claude Code skill 作者 | 人工逐行审查，凭经验判断（不可规模化，质量不稳定） |
| 新 skill 从零开始，缺少脚手架和模板 | 每次创建新 skill（估算: 月均 1-2 次） | 估算 1-2 小时搭建基础结构（基于作者体感） | 同上 | 从已有 skill 复制修改（引入残留内容，遗漏必要结构） |
| skill 修改后缺乏验证手段，不知道改坏了没有 | 每次 skill 修改（估算: 周均 3-5 次） | 发现问题时已在实际使用中出错，回溯成本高 | 同上 | 实际使用中发现问题（反馈周期长，成本高） |

### 目标用户

| 角色 | 典型场景 | Before | After | 预估提效 |
|------|---------|--------|-------|---------|
| Claude Code skill 作者 | 开发和维护自己的 skill 工具集 | 创建靠复制、重构靠手工、验证靠实际使用出错后修复 | `/refactor skill` 自动压缩重构、`/refactor create` 脚手架生成、`/refactor test` 结构验证 | 待验证（预期: 单次重构从 30-60 分钟降到 5-10 分钟） |

### 竞品对比

| 方案 | 定位 | 目标用户 | 核心功能 | 优势 | 局限 |
|------|------|---------|---------|------|------|
| **skill-wrapper** | 面向 Claude Code skill 作者的开发工具集 | skill 作者 | skill 重构压缩、脚手架创建、结构验证 | 专为 skill 文档设计，理解 SKILL.md 结构规范 | 仅覆盖 skill 开发场景，依赖 Claude Code plugin 机制 |
| 手工编辑 | 通用文本编辑 | 所有人 | 无限制的自由编辑 | 零学习成本，完全灵活 | 无质量保障，无结构验证，重复劳动 |
| Claude Code /simplify | 内置代码简化 skill | Claude Code 用户 | 代码审查和简化 | 内置无需安装 | 面向代码而非 skill 文档，不理解 SKILL.md 结构 |

### 汇总

| 版本 | 核心方向 | 核心指标变化 | 状态 | 周期 | 里程碑 |
|------|---------|------------|------|------|--------|
| v1 — 工具闭环 | 建立创建+测试+重构三环节工具链 | ↑ 子命令 1→3；↑ 成功案例 0→2 | 开发中 | 2026.04.16 - 待定 | M1-M3 |

### 版本详情

#### v1 — 工具闭环

- **战略意图**: 验证核心假设——标准化的 skill 开发工具能否显著提升 skill 质量和开发效率
- **投入产出**: 投入估算 3-5 天 → 建立 3 个子命令覆盖 skill 开发全生命周期
- **优先级依据**: 已有 refactor skill 子命令验证了重构可行性，补齐创建和测试即可闭环；无外部依赖
- **风险与依赖**: 风险：skill 结构规范尚未标准化，验证规则可能需要随实践迭代
- **成功指标**: 3 个子命令可用，用 refactor skill 重构 ≥2 个真实 skill 至成熟可用版本并获得正向反馈
- **核心价值**: 1. 首次可以用 `/refactor create` 脚手架生成符合规范的 skill 骨架 2. 首次可以用 `/refactor test` 验证 skill 结构完整性 3. refactor skill 从可用打磨至稳定可靠
- **用户覆盖**: 作者 dogfood
- **核心指标**（基线 → v1）:

| 指标 | 基线 | v1 目标 | 变化 | 来源 |
|------|------|---------|------|------|
| 子命令数量 | 1（skill） | 3（skill, create, test） | ↑ 2 | 目标值，待验证 |
| 重构成功案例 | 0 | ≥2 个真实 skill | ↑ 2 | 目标值，待验证 |
| skill 结构验证规则 | 0 | — | — | 目标值，待验证（随实践确定规则数量） |

# skill-wrapper 产品路线图

## 1. 产品愿景

### 产品核心

- **定位**: 面向 Claude Code skill 作者的质量保障工具
- **动机**: skill 文件是 AI 的行为指令，但指令质量无法验证——模糊表达、隐式规则、缺量化标准导致 AI 执行偏离预期，作者只能在实际使用中才能发现问题
- **长远愿景**: skill 作者在发布前就能验证"AI 会按我的预期执行"——像单元测试保障代码质量一样，lint + simulate 保障指令质量

### 价值体系

| 层级 | 价值 | 衡量指标 |
|------|------|---------|
| **即时价值** | 每次修改 skill 后可立即检查指令缺陷，不再依赖实际使用出错才发现 | lint 有效检出率（目标值，待验证: ≥3 条/skill） |
| **累积价值** | lint 规则集随实践迭代，覆盖的缺陷模式越来越全 | 待验证（预期: 规则数量随使用增长） |
| **战略价值** | skill 质量从"写完就用"升级为"验证后发布"，AI 行为可预测性提升 | 无数据（尚无量化手段） |

### 核心问题

| 问题 | 发生频率 | 单次成本 | 影响面 | 现有应对 |
|------|---------|---------|--------|---------|
| skill 指令模糊导致 AI 执行偏离预期 | 每次 skill 修改后（估算: 周均 3-5 次，基于作者经验） | 实际使用中发现后返工修改（估算: 15-30 分钟，基于作者体感） | 所有 Claude Code skill 作者 | 在实际使用中发现问题（反馈周期长，成本高） |
| 不知道 AI 拿到 skill 后会怎么执行 | 每次发布新 skill 或大改后 | 行为偏离后需多次调试指令措辞（估算: 30-60 分钟） | 同上 | 靠经验猜测 + 试错（不可规模化） |

### 目标用户

| 角色 | 典型场景 | Before | After | 预估提效 |
|------|---------|--------|-------|---------|
| Claude Code skill 作者 | 编写/修改 skill 后验证质量 | 写完即用，实际使用中发现 AI 执行偏离再修改（估算: 2-3 轮试错） | `/skill-wrapper lint` 立即检出指令缺陷，`/skill-wrapper simulate` 预览 AI 行为链 | 待验证（预期: 试错轮次从 2-3 次降到 0-1 次） |

### 竞品对比

| 方案 | 定位 | 目标用户 | 核心功能 | 优势 | 局限 |
|------|------|---------|---------|------|------|
| **skill-wrapper** | 面向 Claude Code skill 作者的质量保障工具 | skill 作者 | 指令缺陷扫描（lint）、行为链验证（simulate） | 专为 AI 指令质量设计，规则来自实战经验 | 仅覆盖 skill 场景，lint 规则需持续迭代 |
| 手工审查 | 人工逐行检查 | 所有人 | 无限制 | 零学习成本 | 依赖经验，不可规模化，容易遗漏 |
| Claude Code /simplify | 内置代码简化 | Claude Code 用户 | 代码审查和简化 | 内置无需安装 | 面向代码，不理解 skill 指令质量 |

### 汇总

| 版本 | 核心方向 | 核心指标变化 | 状态 | 周期 | 里程碑 |

### 版本详情

#### v1 — 质量闭环

- **战略意图**: 验证核心假设——基于规则的指令缺陷扫描和行为链模拟能否在 skill 发布前暴露 AI 执行偏离风险
- **投入产出**: 投入估算 3-5 天 → 建立 lint（10 条规则）+ simulate（行为链 dry-run）
- **优先级依据**: lint 可立即验证价值（扫描已有 skill），simulate 需要 lint 规则集作为基础；无外部依赖
- **风险与依赖**: 风险：lint 规则覆盖面不足（需在实践中迭代）；simulate 对复杂 skill 的行为链推理准确度未知
- **成功指标**: lint 在 ≥2 个真实 skill 上检出 ≥3 条有效指令缺陷（作者确认影响 AI 执行），simulate 行为链与实际执行一致
- **核心价值**: 1. 首次可以在发布前检查 skill 指令缺陷 2. 首次可以预览 AI 拿到 skill 后的行为链
- **用户覆盖**: 作者 dogfood
- **核心指标**（基线 → v1）:

| 指标 | 基线 | v1 目标 | 变化 | 来源 |
|------|------|---------|------|------|
| lint 规则数 | 0 | 10（L1-L10） | ↑ 10 | 目标值，待验证 |
| 有效检出 | 0 | ≥3 条/skill | ↑ 3 | 目标值，待验证 |
| simulate 覆盖 | 0 | stage+step+gate 级别 | — | 目标值，待验证 |
